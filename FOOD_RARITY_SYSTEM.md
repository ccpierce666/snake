# 食物等级与稀有度系统

## 系统概述

全新的食物系统基于 **2的倍数等级**（2, 4, 8, 16, 32, 64, 128, 256, 512, 1024, 2048），每个等级都有独特的颜色、大小和生成概率。

## 颜色设计

### 颜色渐进体系

**绿色→黄色→橙色→红色→蓝色→紫色**

这形成了一个视觉递进：
- **绿色**：常见，安全的基础食物
- **黄色/橙色**：中等等级，有价值
- **红色**：高级食物，很珍贵
- **蓝色/紫色**：极端稀有，传奇级别

## 游戏体验分析

### 早期（主要等级1-3）
- 玩家频繁看到绿、黄、黄色食物
- 增长速度：2、4、8段（倍数调整后可能是4、8、16）
- 感受：稳定增长，学习基础

### 中期（主要等级4-5）
- 开始看到橙色食物
- 增长速度：16、32段
- 感受：增长速度明显提升，激励感增强

### 后期（偶见等级6）
- 红色食物极为珍贵
- 增长速度：+64段
- 感受：发现红色食物时有惊喜感

### 极端后期（等级7+）
- 蓝色/紫色食物几乎看不到
- 增长速度：+128及以上
- 感受：极为珍稀，有的玩家可能玩了很久都没见过

## 游戏平衡考虑

### 为什么使用2的倍数？
- **数学优美**：2的倍数形成完美的指数级数
- **平衡性好**：避免了随意的数值，符合游戏设计原则
- **易于理解**：玩家看到2、4、8、16等数字会有熟悉感

### 为什么等级6开始概率很小？
- **稀有度梯度**：前5个等级累计94%的概率，剩下6%给等级6及以上
- **保持新鲜感**：高等级食物的出现提供惊喜感
- **游戏寿命**：防止玩家过快获得超大增长

### 为什么最高是等级11（2048）？
- **终极目标**：2048是个象征性的数字（2^11）
- **防止失衡**：避免无限增长
- **够长的尾巴**：11个等级提供充足的成长阶梯

## 调整建议

### 如果觉得高等级食物太稀有
降低等级6及以上的阈值：
```lua
elseif rand < 0.12 then value = 6      -- 从0.06改为0.12，概率翻倍
```

### 如果觉得基础增长太快
修改增长倍数（与非线性增长系统配合）：
```lua
-- 使用更小的倍数
local scaledGrowth = math.floor(baseGrowth * growthMultiplier * 0.5 + 0.5)
```

### 如果想要更多等级
可以扩展到等级12（4096），但需要调整概率：
```lua
if rand < 0.00005 then value = 12      -- 新增最高等级
```
