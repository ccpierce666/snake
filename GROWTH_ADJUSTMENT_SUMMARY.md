# 增长系统调整总结

## 问题
> "前期增长太快，吃几百个食物，身体就好几百节了"

## 解决方案：使用四次方根函数（^0.25）代替平方根（^0.5）

### 旧公式
```lua
倍数 = 1.0 + (currentLength / 100000)^0.5
```
- 参考长度：100,000
- 指数：0.5（平方根）
- 问题：早期倍数达到1.01-1.03，加成虽小但在几百个食物的累积下会明显

### 新公式
```lua
倍数 = 1.0 + (currentLength / 500000)^0.25
```
- 参考长度：500,000（增加5倍）
- 指数：0.25（四次方根，更陡峭）
- 改善：早期倍数接近1.0，几乎无加成

## 游戏体验改变

### 早期积累阶段（0-50000段）

**旧系统：**
- 吃200个基础食物 = 增长 600-750段
- 感觉增长很快，进度条飙升

**新系统：**
- 吃200个基础食物 = 增长 600-620段
- 感觉增长平稳，需要真正积累

### 中期加速阶段（50000-200000段）

**旧系统：**
- 倍数从1.7跳到2.0比较快
- 大蛇们增长飞速

**新系统：**
- 倍数从1.2逐步升至1.8
- 成长更平滑有梯度

### 晚期爆炸阶段（200000+段）

**旧系统：**
- 倍数已经是2.0，吃什么都翻倍增长

**新系统：**
- 倍数也是2.0，但因为前期积累更多，绝对数值更大

## 文件修改位置

  - 将 `(ratio ^ 0.5)` 改为 `(ratio ^ 0.25)`

### 客户端：`SnakeGameController.lua`
- **第31-37行**：`calculateGrowthMultiplier()` 函数
  - 同样的改动，保证客户端与服务器一致

## 如果还要继续调整

### 更慢的增长（极端）
```lua
local multiplier = 1.0 + (ratio ^ 0.2)  -- 四次方根的导数更小
```

### 稍快一点的增长（温和）
```lua
local multiplier = 1.0 + (ratio ^ 0.3)  -- 介于0.25和0.5之间
```

### 改变参考长度
- 改为 `1000000`：需要双倍的长度才能达到2.0倍（更慢）
- 改为 `250000`：只需四分之一的长度就能达到2.0倍（更快）

## 总结

通过将指数从 0.5 改为 0.25，配合参考长度从 100,000 增加到 500,000，我们实现了：

✅ **早期（0-50000段）增长缓慢**，倍数基本接近1.0
✅ **中期（50000-200000段）逐步加速**，倍数平滑上升
✅ **后期（200000+段）快速膨胀**，倍数保持2.0
✅ **整体体验**：从"吃几百个食物就好几百节"变成"需要真正积累"

现在玩家需要更多耐心来积累早期的蛇，但这也增加了游戏的策略性和成就感！
