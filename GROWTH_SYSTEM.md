# 非线性增长倍数系统（Accelerated Growth System）

## 核心递推公式

```
增长倍数 = 1.0 + (当前长度 / 100000)^0.5
```

其中：
- 当前长度：蛇的身体段数
- 100000：参考长度（达到最大倍数的目标）
- ^0.5：平方根函数（平滑加速曲线）
- 倍数范围：1.0 到 2.0

## 为什么采用平方根函数？

1. **非线性增长**：不是简单的线性增长
2. **平缓起始**：小蛇早期增长慢（倍数≈1.0）
3. **加速中期**：中期开始加速（倍数≈1.1~1.5）
4. **收敛上限**：大蛇增长快但稳定（倍数→2.0）
5. **防止无限膨胀**：上限2.0防止超大蛇变得过于庞大

## 游戏阶段分析

### 第一阶段：早期探索（0-1000段）
- **增长倍数**：1.00 → 1.032
- **特征**：增长最慢，需要大量食物累积
- **目的**：新手学习阶段，有足够时间熟悉操作
- **玩家体验**：吃了很多食物，身体变化才明显

### 第二阶段：中期发展（1000-100000段）
- **增长倍数**：1.032 → 1.414
- **特征**：逐步加速，倍数线性上升
- **目的**：维持游戏节奏，让进度不断加快
- **玩家体验**：吃食物的收益逐步提高，越来越有成就感

### 第三阶段：晚期统治（100000+段）
- **增长倍数**：固定 2.0
- **特征**：增长最快，但身体大小不再变化
- **目的**：超大蛇快速扩展版图，主导游戏
- **玩家体验**：身体长度飙升，但视觉上不再变大

## 递推逻辑详解

### 平方根函数的妙用

使用 `ratio^0.5` 而不是线性的 `ratio` 的原因：

```
线性（ratio）:      倍数从 1.0 线性增长到 2.0
平方根（ratio^0.5）: 倍数缓慢起始（早期1.0），中期加速，晚期趋缓
```

数学意义：
- f(0) = 1.0（起点）
- f(0.5) ≈ 1.224（中点倍数只有1.2+）
- f(1.0) = 2.0（终点）

这形成一条"S型"加速曲线，非常符合游戏设计理念。

### 四舍五入处理

```lua
local scaledGrowth = math.floor(baseGrowth * growthMultiplier + 0.5)
```

确保增长值始终是整数，避免浮点数造成的视觉差异。

## 实现代码

### 服务器端（SnakeGameService.lua）
```lua
local function calculateGrowthMultiplier(currentLength)
    local maxLength = 100000
    local normalizedLength = math.min(currentLength, maxLength)
    local ratio = normalizedLength / maxLength  -- 0 到 1
    local multiplier = 1.0 + (ratio ^ 0.5)      -- 1 到 2
    return multiplier
end
```

### 应用方式
```lua
local currentLength = #s.body
local growthMultiplier = calculateGrowthMultiplier(currentLength)

for i = #foundFoodList, 1, -1 do
    local foodValue = foodItem.value or 1
    local baseGrowth = growthMap[foodValue]
    local scaledGrowth = math.floor(baseGrowth * growthMultiplier + 0.5)
    totalGrowth = totalGrowth + scaledGrowth
end

s.growthPending = (s.growthPending or 0) + totalGrowth
```

## 调整建议

如果需要改变增长速度，有几个参数可以调整：

### 1. 改变加速曲线
- **更缓慢的加速**：`(ratio ^ 0.6)` 或更高的指数
- **更快速的加速**：`(ratio ^ 0.4)` 或更低的指数
- **建议**：保持在 0.4-0.6 之间

### 2. 改变参考长度
- **更早达到2.0倍**：改为 `50000`
- **更晚达到2.0倍**：改为 `200000`
- **建议**：与身体分级系统保持协调

### 3. 改变上限倍数
- **更温和的增长**：改为 `1.5` 而不是 `2.0`
- **更激进的增长**：改为 `2.5` 或 `3.0`
- **建议**：不超过 2.5，防止游戏失衡

### 示例：温和增长（倍数范围1.0-1.5）
```lua
local multiplier = 1.0 + (ratio ^ 0.5) * 0.5
```

### 示例：激进增长（倍数范围1.0-2.5）
```lua
local multiplier = 1.0 + (ratio ^ 0.5) * 1.5
```

## 与身体分级系统的协调

身体分级系统（Tier 1-5）与增长倍数系统无缝协作：

| Tier | 身体长度 | 倍数范围 | 身体大小 |
|------|---------|---------|--------|
| 1 | 0-99 | 1.00 | 1.0 |
| 2 | 100-999 | 1.01-1.032 | 1.3 |
| 3 | 1000-9999 | 1.032-1.10 | 1.6 |
| 4 | 10000-99999 | 1.10-1.414 | 1.9 |
| 5 | 100000+ | 2.0 | 2.2（固定） |

这个系统设计的核心目的是：
- **保留挑战**：早期增长慢，需要策略和耐心
- **提供快感**：中期加速，让玩家感到进度在加快
- **避免无限膨胀**：晚期固定大小，维持游戏平衡
