# 非线性增长倍数系统 V2（前期极慢增长）

## 核心递推公式（更新）

```
增长倍数 = 1.0 + (当前长度 / 500000)^0.25
```

**关键改动：**
- 参考长度：从 100000 改为 **500000**（增加5倍）
- 指数：从 0.5 改为 **0.25**（更陡峭的曲线）
- 效果：早期增长**极其缓慢**，中后期才开始加速

## 为什么这样调整？

### 旧公式 vs 新公式对比

| 身体长度 | 旧公式(^0.5) | 新公式(^0.25) | 差异 |
|---------|------------|-------------|------|
| 0 | 1.00 | 1.00 | 相同 |
| 100 | 1.01 | 1.0006 | 新公式慢100倍 |
| 1000 | 1.032 | 1.002 | 新公式慢16倍 |
| 10000 | 1.10 | 1.0063 | 新公式慢17倍 |
| 100000 | 1.414 | 1.019 | 新公式慢74倍 |
| 500000 | 2.0（上限） | 2.0（上限） | 相同达到上限 |

### 使用四次方根（^0.25）的意义

- `^1.0`：线性增长（太快）
- `^0.5`：平方根增长（前期还是太快）
- `^0.25`：四次方根增长（前期极慢，中后期加速）
- `^0.1`：更陡峭（太极端）

**四次方根曲线特性：**
```
y = x^0.25 的函数图像

1.0 |                                    ╱
    |                                ╱
倍  |                            ╱
数  |                        ╱
    |                    ╱
1.5 |                ╱
    |            ╱
    |        ╱
    |    ╱
    |╱
1.0 |────────────────────────────────
    0        100000      250000      500000
              身体长度
```

早期几乎看不到增长，到500000时才达到2.0倍。

## 游戏阶段分析

### 第一阶段：超级早期（0-10000段）
- **增长倍数**：1.000 → 1.063
- **特征**：倍数增长几乎看不到，吃几百个food才能变长几百段
- **所需食物**：需要吃大量基础食物才能积累
- **玩家体验**：需要很长时间的积累期，但逐步有成就感

**具体例子：**
- 吃100个value=1的食物：增长 3×100 = 300段（无加成）
- 吃200个value=1的食物：增长 3×200 = 600段（加成<1%）

### 第二阶段：早中期（10000-100000段）
- **增长倍数**：1.063 → 1.333
- **特征**：倍数开始明显上升，增长速度加快
- **目的**：经过初期积累后，开始看到成果
- **玩家体验**：增长速度逐步提升，激励感增加

### 第三阶段：中后期（100000-500000段）
- **增长倍数**：1.333 → 2.0
- **特征**：加速持续，倍数快速上升
- **目的**：大蛇开始快速膨胀
- **玩家体验**：吃食物的效果越来越明显，快速扩展

### 第四阶段：超大蛇（500000+段）
- **增长倍数**：固定 2.0
- **特征**：倍数达到上限，保持双倍增长
- **目的**：防止无限膨胀，维持平衡
- **玩家体验**：已经是绝对主导者，快速扩张但大小不变

## 与前期需求的对应

### 原问题：吃几百个食物就好几百节

**新系统的改善：**
- 吃200个value=1食物（基础增长600）：现在只增长 ~600 段（倍数1.000）
- 在0-10000段阶段，倍数基本接近1.0，几乎没有加成
- 需要累积到50000+段，倍数才会有明显影响

**新的增长节奏：**
```
早期（0-10000段）：
  - 吃100个food → +300段
  - 吃200个food → +600段
  - 吃500个food → +1500段
  需要吃很多食物才能达到10000段

中期（10000-100000段）：
  - 吃100个food → +318-350段（有6-16%加成）
  - 吃200个food → +636-700段
  - 吃500个food → +1590-1750段

后期（100000+段）：
  - 吃100个food → +400-600段（有33-100%加成）
  - 吃200个food → +800-1200段
  - 吃500个food → +2000-3000段
```

## 实现代码

### 服务器端
```lua
local function calculateGrowthMultiplier(currentLength)
    local maxLength = 500000
    local normalizedLength = math.min(currentLength, maxLength)
    local ratio = normalizedLength / maxLength
    local multiplier = 1.0 + (ratio ^ 0.25)
    return multiplier
end
```

### 应用方式保持不变
```lua
local currentLength = #s.body
local growthMultiplier = calculateGrowthMultiplier(currentLength)
local scaledGrowth = math.floor(baseGrowth * growthMultiplier + 0.5)
```

## 可进一步调整的选项

如果觉得还是太快，可以继续调整：

### 选项1：使用 ^0.2 而不是 ^0.25（更极端）
```lua
local multiplier = 1.0 + (ratio ^ 0.2)  -- 增长更慢
```

### 选项2：增加参考长度到 1000000
```lua
local maxLength = 1000000  -- 需要更长的游戏时间
```

### 选项3：改为 1.5x 而不是 2.0x（上限不那么高）
```lua
local multiplier = 1.0 + (ratio ^ 0.25) * 0.5  -- 范围1.0-1.5
```

## 总结

新公式通过以下方式解决"前期增长太快"的问题：

✅ **大幅降低早期倍数**：0-10000段倍数 < 1.1（几乎无加成）
✅ **延长积累期**：需要吃更多食物才能有明显变化
✅ **保留中后期加速**：中后期仍然有快速增长的快感
✅ **防止失衡**：上限依然是2.0，保持游戏平衡
✅ **更自然的进度**：早期慢速探索，后期快速膨胀
