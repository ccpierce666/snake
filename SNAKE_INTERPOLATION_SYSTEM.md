# 蛇身体渲染系统 V3 (插值算法)

## 核心改进：线性插值 (Linear Interpolation)

为了彻底解决“跳点采样”导致的缝隙问题，我们引入了**线性插值算法**。

### 问题回顾
旧的采样方法是：
- 遍历服务器发来的 `body` 点
- 如果 `dist(当前点, 上个渲染点) >= spacing`，就渲染当前点

缺陷：
- 假设 `spacing = 0.3`
- 服务器点间距 `0.25`
- 点1(0) -> 渲染
- 点2(0.25) -> 跳过 (0.25 < 0.3)
- 点3(0.50) -> 渲染 (0.50 >= 0.3)
- **实际间距变成了 0.50！** 远大于想要的 0.3，导致巨大缝隙。

### 新的插值算法
不再依赖服务器点的原始位置，而是把它们连成一条折线，在这条线上**均匀撒点**。

逻辑：
1. 累积走过的距离 `accumulatedDist`
2. 每当累积距离超过 `spacing`，就在当前线段上计算出精确位置插入一个球
3. 扣除 `spacing`，继续累积

效果：
- 无论服务器点间距是0.25、0.5还是0.1
- 渲染出来的球体间距**恒定为** `spacing`
- 彻底消除了因离散数据导致的间距误差

## 参数设置

### 身体大小
- **初生蛇 (长度 < 10)**：0.33 (1/3格子)
- **成长蛇 (长度 ≥ 10)**：0.66 (2/3格子) - **直接增长一倍**

### 渲染间距
- **Spacing = bodySize × 0.8**
- 20% 的重叠度，确保绝对紧密，即使在拐弯处也不会有缝隙

### 头部大小
- **HeadSize = bodySize × 1.2**

## 验证
- ✅ 0.33大小的蛇，间距严格为 0.264
- ✅ 0.66大小的蛇，间距严格为 0.528
- ✅ 无论怎么转弯、伸长，球体始终均匀紧密排列

现在，蛇的身体将像一条真正的绳子一样连续，不再有任何断裂感。
