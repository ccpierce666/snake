# 蛇身体大小 - 压缩版本（增长变慢）

## 调整说明

用户反馈：身体增长太快，不到1K就很长了。已调整为**压缩版本**，身体增长极为缓慢。

## 新的大小映射表

### 详细的长度 vs 大小对照

| 身体长度 | 身体大小 | 变化 | 说明 |
|---------|---------|------|------|
| **1-50** | 0.33 | 固定 | **超长时间保持1/3格子** |
| **100** | 0.39 | +0.06 | 100个食物后才微微扩大 |
| **500** | 0.60 | +0.27 | **900个食物后达到0.6** |
| **1,000** | 0.67 | +0.07 | 缓慢扩大 |
| **5,000** | 0.80 | +0.13 | 约0.8 |
| **10,000** | 0.85 | +0.05 | 接近0.85 |
| **50,000** | 0.95 | +0.10 | **约1格** |
| **100,000** | 1.14 | +0.19 | 开始放大 |
| **500,000** | 1.80 | +0.66 | **达到最大** |
| **1,000,000+** | 1.80 | 固定 | 上限，防止失衡 |

## 新旧对比

### 早期阶段对比

| 长度 | 旧大小 | 新大小 | 变化 |
|------|--------|--------|------|
| 10 | 0.50 | 0.33 | **保持1/3格子更久** |
| 50 | 0.60 | 0.33 | **仍是1/3格子** |
| 100 | 0.70 | 0.39 | **增长减半** |
| 500 | 0.86 | 0.60 | **增长大幅减缓** |
| 1,000 | 0.93 | 0.67 | **增长继续减缓** |

### 为什么要压缩？

```
旧系统问题：
- 长度100：0.70大小（已经很明显）
- 长度1000：0.93大小（接近1格，太快了）
- 用户感受：不到1K就很长，没有积累感

新系统改善：
- 长度100：0.39大小（还很小）
- 长度1000：0.67大小（缓慢接近）
- 长度50000：0.95大小（才接近1格）
- 用户感受：需要长时间积累，更有成就感
```

## 分段详解

### 阶段1：超早期（长度 0-50）
```
大小：0.33（固定）
时长：吃约150-200个基础食物
特征：蛇保持1/3格子大小，极小
体验：起始阶段，需要大量积累
```

### 阶段2：早期（长度 50-500）
```
大小：0.33 → 0.60（线性增长）
时长：再吃约250-300个食物
速度：每100食物增长0.06
特征：非常缓慢地扩大，变化不明显
体验：耐心积累期，感受到缓慢成长
```

### 阶段3：初中期（长度 500-50,000）
```
大小：0.60 → 0.95（缓慢增长）
时长：需要吃非常多食物
速度：每10,000食物增长0.07
特征：长时间停留在1格左右
体验：长期积累，最终达到1格目标
```

### 阶段4：中后期（长度 50,000-500,000）
```
大小：0.95 → 1.80（加速增长）
时长：长期的成长阶段
速度：每10,000食物增长0.002
特征：终于开始明显变大
体验：大蛇们的舞台
```

### 阶段5：极后期（长度 500,000+）
```
大小：1.80（固定）
特征：达到视觉上限
目的：防止过度膨胀
体验：绝对统治者
```

## 数值设计理由

### 为什么第一阶段是50而不是10？
- 旧的10太短，扩大太快
- 50给了更长的初始积累期
- 玩家能有足够的"小蛇"体验

### 为什么第二阶段到500而不是100？
- 100-500只增长0.27，变化极小
- 这给了玩家约450个食物的"准备期"
- 不会在100就明显看到变化

### 为什么到50,000才接近1格？
- 旧系统在10,000就到1.0
- 新系统在50,000才到0.95
- **增长减缓5倍**

### 为什么上限改为1.8而不是2.5？
- 防止后期蛇变得太大
- 1.8已经足够有视觉冲击
- 保持游戏平衡性

## 代码实现

```lua
local function calculateBodySize(currentLength)
    if currentLength < 50 then
        return 0.33
    elseif currentLength < 500 then
        local ratio = (currentLength - 50) / 450
        return 0.33 + ratio * 0.27
    elseif currentLength < 50000 then
        local ratio = (currentLength - 500) / 49500
        return 0.6 + ratio * 0.35
    elseif currentLength < 500000 then
        local ratio = (currentLength - 50000) / 450000
        return 0.95 + ratio * 0.85
    else
        return 1.8
    end
end
```

## 总结

压缩版本大幅延长了早期增长时间：

✅ **超长期保持1/3格子**：长度50内保持不变
✅ **极缓慢的扩大**：长度500前基本看不出变化
✅ **到达1格目标变远**：从10K改为50K
✅ **更低的最终上限**：1.8而非2.5，防止过大
✅ **更长的成就周期**：玩家能感受到更长的成长过程

现在蛇的增长变得**极为缓慢**，玩家需要更长时间的积累才能看到明显变化！
