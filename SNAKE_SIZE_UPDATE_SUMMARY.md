# 蛇身体动态大小系统 - 更新总结

## 用户反馈与改进

### 原始问题
> "蛇的变化可以再明显一点，可以一开始不到一个格子（1/3格子）身长个位数以内，因为玩家后面的身长数会集中在几k，几十k以内，这个时候可以占1格左右，后面到了几百k，10m的时候在放大"

### 核心需求分析
1. **早期阶段**：蛇极小（1/3格子），身长个位数
2. **中期阶段**：蛇稳定（1格左右），身长几k-几十k
3. **后期阶段**：蛇逐步放大（1.0-2.5+），身长几百k-10m

## 实现方案

### 从固定等级 → 动态计算

**旧系统（弃用）：**
- 5个固定Tier等级
- 突兀的跳跃式增长
- 无法精细调控

**新系统（采用）：**
- 动态计算函数 `calculateBodySize(currentLength)`
- 平滑的线性插值
- 完美对应身体长度

## 详细的大小映射

```
身体长度    →    身体大小    →    占比
  0-9              0.33         1/3格子     ← 极早期
  10-100           0.50~0.70    1/2~2.1倍   ← 早期扩大
  100-10,000       0.70~1.00    2.1~3.0倍   ← 初中期稳定
  10,000-100,000   1.00~1.50    3.0~4.5倍   ← 中期放大
  100,000-1,000,000 1.50~2.50   4.5~7.5倍   ← 后期快速放大
  1,000,000+       2.50         7.5倍       ← 极后期固定
```

## 关键参数表

### 分界点与增长量

| 分段 | 长度范围 | 大小范围 | 增长量 | 速率 |
|------|---------|---------|--------|------|
| 1 | 0-10 | 0.33 固定 | - | - |
| 2 | 10-100 | 0.50→0.70 | 0.2 | 快 |
| 3 | 100-10K | 0.70→1.00 | 0.3 | 较快 |
| 4 | 10K-100K | 1.00→1.50 | 0.5 | 中等 |
| 5 | 100K-1M | 1.50→2.50 | 1.0 | 快 |
| 6 | 1M+ | 2.50 固定 | - | - |

### 线性插值公式

```lua
新大小 = 起始大小 + ((当前长度 - 起始长度) / 分段长度) × 增长量
```

**例子：长度50,000时**
```
属于第4段：10K-100K
ratio = (50000 - 10000) / 90000 = 0.4444
size = 1.0 + 0.4444 × 0.5 = 1.2222
```

#### 改动1：新增 `calculateBodySize()` 函数（L31-63）
```lua
local function calculateBodySize(currentLength)
    if currentLength < 10 then
        return 0.33
    elseif currentLength < 100 then
        local ratio = (currentLength - 10) / 90
        return 0.5 + ratio * 0.2
    -- ... 更多分段
    end
end
```

## 性能分析

### 计算成本
- **时间复杂度**：O(1)（常数时间）
- **空间复杂度**：O(1)（无额外存储）
- **每帧调用**：1次（Heartbeat回调）

### 对比旧系统
```
旧系统（Tier查询）：
  - 遍历BODY_SIZE_TIERS数组（5次迭代）
  - 条件判断（多个if-elseif）
  
新系统（动态计算）：
  - 直接数学计算（乘除）
  - 单层if-elseif结构
  
结论：性能相同或更优
```
