# 蛇身体紧密贴合修复说明

## 问题诊断

用户反馈：球体变小了，但球体之间还是有缝隙，看起来不像一条蛇。

### 根本原因

我之前的理解有误。问题不仅仅是"头部大小"的问题，而是一个更深层的采样逻辑问题：

1. **服务器逻辑**：每一帧，蛇头按照 `SNAKE_SPEED * GAME_TICK` 的速度移动，并将新头部位置插入到 `body` 数组的开头。
   ```lua
   local newHead = head + s.direction * SNAKE_SPEED * GAME_TICK
   table.insert(s.body, 1, newHead)
   ```
   
2. **服务器身体数据**：`body` 数组中相邻两个位置点之间的距离大约是 `SNAKE_SPEED * GAME_TICK`（固定的）。

3. **客户端显示**：客户端从 `body` 数组中采样位置，然后在每个位置渲染一个球体。

### 问题的真正根源

**错误的思路**：
```
我只改变了球的大小（从1.0到0.33），但没有改变采样间距
结果：
  - 球的大小：0.33
  - 采样间距：保持原来的间距（比如1.0）
  - 结果：0.33大小的球，相隔1.0的距离 → 有明显缝隙！
```

**正确的思路**：
```
球的大小变化了，采样间距也要相应变化
  - 球的大小：0.33
  - 采样间距：0.33 * 0.9 = 0.297
  - 结果：0.33大小的球，相隔0.297的距离 → 紧密贴合！
```

## 解决方案

### 核心改动

修改 `addSnakeRenderPoints` 函数，使采样间距与身体大小**成比例**：

```lua
-- 采样间距 = bodySize * 0.9
-- 这样无论bodySize是0.33、1.0还是1.8，球体之间的比例关系都是一致的
local samplingDistance = bodySize * 0.9

for i = 2, #body do
    local dist = (body[i] - lastPos).Magnitude
    if dist >= samplingDistance then
        -- 采样这个位置
        table.insert(snakePositions, {...})
        lastPos = body[i]
    end
end
```

### 为什么是0.9而不是1.0？

- **0.9** = 10%的重叠，球体之间紧密贴合，看起来像一条蛇
- **1.0** = 完全相接，刚好接触，视觉上可能有缝隙感
- **0.8** 以下 = 过度重叠，看起来拥挤不自然

## 数值对应关系

### 早期蛇（长度<50，bodySize=0.33）

```
samplingDistance = 0.33 * 0.9 = 0.297

body数组（服务器数据）：
位置1  位置2  位置3  位置4  位置5  ...
  |      |      |      |      |
0.5    0.5    0.5    0.5    0.5     (相邻位置之间约0.5)

客户端采样（samplingDistance=0.297）：
if dist >= 0.297:  (✓ 0.5 >= 0.297)
    采样 → 球1(0.33大小) → 位置2
if dist >= 0.297:  (✓ 0.5 >= 0.297)
    采样 → 球2(0.33大小) → 位置4
...

结果：每隔几个body位置采样一次，确保球体紧密贴合
```

### 中期蛇（长度500-1000，bodySize≈0.65）

```
samplingDistance = 0.65 * 0.9 = 0.585

body数组（相邻位置仍然约0.5）：
if dist >= 0.585:  (✗ 0.5 < 0.585)
    不采样 → 跳过这个位置

这意味着球越大，采样越稀疏，但每个球都更大，
所以视觉上球体之间仍然紧密贴合（不浪费采样点）
```

### 后期蛇（长度500000+，bodySize=1.8）

```
samplingDistance = 1.8 * 0.9 = 1.62

body数组（相邻位置仍然约0.5）：
if dist >= 1.62:  (✗ 0.5 < 1.62)
    不采样频繁，采样点更稀疏
    但每个球都是1.8大小，所以球体覆盖的范围更大
```

## 关键洞察

### 采样距离与球体大小的关系

```
最终蛇的视觉紧密度 ∝ (球体大小 / 采样距离)

旧系统：
  早期：(0.33 / 固定距离X) → 可能有缝隙
  后期：(1.8 / 固定距离X) → 可能没有缝隙

新系统：
  早期：(0.33 / 0.297) ≈ 1.11 → 紧密贴合
  后期：(1.8 / 1.62) ≈ 1.11 → 紧密贴合
  
比例一致！所以无论蛇多大，都能保持一致的视觉效果
```

## 代码改动对比

### 旧代码的问题
```lua
local spacing = bodySize  -- ✗ 这是错的
-- spacing是用来过滤body数组中的位置的
-- 但body数组中相邻位置的实际距离是服务器决定的（约SNAKE_SPEED*GAME_TICK）
-- 所以这个spacing无法真正控制最终渲染的球体之间的距离
```

### 新代码
```lua
local samplingDistance = bodySize * 0.9  -- ✓ 正确
-- 这个距离用来从body数组中采样
-- 采样稀疏度（bodySize越大，采样间距越大）
-- 所以即使球变大了，采样也变稀疏，最终球体之间的距离仍然保持一致的比例
```

## 总结

**修复的本质**：
- 不是简单地改变球的大小
- 而是让采样间距与球的大小**成比例变化**
- 这样无论蛇的身体多大或多小，球体之间的相对间距都保持一致
- 最终呈现出一条始终紧密连贯的蛇

**数学关系**：
```
球体之间的紧密度 = (球体大小) / (采样间距)
                = bodySize / (bodySize * 0.9)
                = 1 / 0.9
                ≈ 1.11（常数，与bodySize无关！）
```

所以现在蛇从始至终都能保持一致的、紧密的视觉效果。
